# LiquidMotionSPA

ビルドレス Vanilla JS + Canvas 2D のリキッドモーション玩具。`python3 -m http.server` で配信して動作確認する(ヘッドレス検証は Playwright + `window.__fluid` フック)。

## 絶対に守る仕様(過去に退行した点)

### ノズルの役割 — 滴下と受けのふたつ

各トレイ(上下リザーバ)にはノズルがちょうど2個あり、**役割が異なる**:

- **滴下ノズル(drip)**: リザーバの液を排出する唯一の出口。
- **受けノズル(catch)**: 対面トレイの滴下ノズルから落ちてくる液を受けるだけの入口。**リザーバに溜まった液がここから漏れ出てはいけない。**
- 容器を反転(フリップ)すると使われるノズルが入れ替わる: 新しい上側トレイの滴下ノズルが排出し、その液は点対称の位置(`xCatch = w - xDrip`)にある対面トレイの受けノズルに落ちる。

この役割分担は**バルブやスクリプト処理で実現してはならない**(下記参照)。`js/level.js` ではトレイの高さだけで実現している: トレイ全体が滴下ノズルへ向かって下る一定勾配を持ち、滴下ノズルの開口がトレイの最低点、受けノズルの開口が最高点。静止液は常に受けノズルから離れ、滴下ノズルからのみ流下する。

### 素直な流体計算 — 人工処理の禁止

流量・流速は開口の幅、重力、粒子物性(反発・凝集・摩擦)から創発的に決まる。以下を再導入しないこと:

- 粒子速度の直接書き換えによる強制加速(旧 `throttleNozzles` の ~300px/s 引き抜き)
- 一方通行バルブ、位置スナップ、吸引、横ドリフトなどの位置直接操作
- タイマーによるスクリプト点滴(一定間隔で1滴ずつ等)

### その他の不変条件

- 液体は**単色**(生成ごとにランダムな1色相、`Fluid.hue`)。凝集は全粒子ペアに適用。
- ステージ(トレイ開口・段差・歯車・シーソー)は `buildLevel()` の**手続き生成**で、ページ読込ごと/リセットボタンで毎回変わる。
- レイアウト全体は点対称 `(x,y) → (w-x, h-y)`。反転しても同じ遊びが成立するための前提なので崩さない。

## 構成

- `js/physics.js` — Verlet 粒子 + 反発/凝集 + カプセル衝突。周囲媒質は浮力+粘性抵抗として作用。
- `js/level.js` — ステージの手続き生成(トレイ・ノズル・段差・ロータ配置)。
- `js/rotor.js` — 歯車(Wheel)とシーソー(Seesaw)。液の衝突・重みで駆動。
- `js/renderer.js` — メタボール風描画。
- `js/main.js` / `js/orientation.js` — ループ、傾きセンサ→重力、反転・リセットUI。
